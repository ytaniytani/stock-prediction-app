# 日経225先物 短期売買サポートWebアプリ — 仕様書 v0.1

> 📌 実装状況: `app/` にReact+TypeScript製の実装あり（P1〜P4の主要機能＋8章の追加提案を一通り実装、詳細は `app/README.md`）。
> ネットワークポリシー上この開発環境からは証券データ配信元に直接アクセスできないため、実データはCSVインポート方式を正式な入力経路とした（4章参照）。初回起動時は架空のサンプルデータで動作確認できる。

短期売買（デイトレード〜数日スイング）の判断材料として、過去約30年の市場データから
**「いまと似た状況のとき、翌日・当日に上がった確率は何%だったか」** を提示するWebアプリ。

> ⚠️ **本アプリの位置づけ（最重要）**
> 本アプリは**過去データの統計的な集計結果を表示する参考ツール**であり、将来の値動きを保証するものではない。
> 投資助言・売買推奨ではなく、最終的な売買判断と責任は利用者に帰属する。
> この免責は全画面のフッターに常時表示する。

---

## 1. 目的とコンセプト

- **対象商品**: 日経225先物（ラージ/ミニ。日中セッション＋ナイトセッション）
- **利用シーン**: 「今買うべきか？」「今日の引けまでに買うべきか？」「明日の寄りで買うべきか？」を、根拠付きで確認する
- **コアの価値**: 「5日連続で合計20%下がった翌日に上がった確率は98.5%（過去30年で67回中66回）」のように、
  **条件付き確率＋サンプル数＋実際の事例**をセットで見せること
- **設計思想**: ブラックボックスのAI予測ではなく、**「何を根拠にその数字が出たか」を全て開示する統計エンジン**を核にする。
  利用者が根拠を検証できることを最優先する

## 3. 機能仕様

### 3.1 メインダッシュボード「今・今日・明日」

3つの時間軸それぞれについて、**買い方向の優位性スコア**を表示する。

```
┌─────────────────────────────────────────────┐
│  今買う？     [やや優位 62%]  n=142  ▲根拠を見る │
│  今日買う？   [中立   51%]  n=310  ▲根拠を見る │
│  明日買う？   [優位   68%]  n= 89  ▲根拠を見る │
└─────────────────────────────────────────────┘
```

- **今（Now）**: 現時点の状況（前夜の米国市場・為替・ナイトセッション・当日ここまでの値動き）に一致する過去事例で、その後引けまでに上昇した確率
- **今日（Today）**: 当日寄り〜引けの上昇確率（寄り前は前夜データのみで判定）
- **明日（Tomorrow）**: 当日引け時点の状況から、翌営業日の上昇確率
- 各スコアには必ず **サンプル数 n・上昇時平均リターン・下落時平均リターン・最大ドローダウン** を併記する
- 「根拠を見る」を開くと、マッチした条件・過去の該当事例一覧（日付・当時のチャート）を表示する

#### スコアの表示ルール

| 上昇確率 | 表示 | 条件 |
|---|---|---|
| 65%以上 | 優位（緑） | n ≥ 30 |
| 55〜65% | やや優位 | n ≥ 30 |
| 45〜55% | 中立 | — |
| 35〜45% | やや劣位 | n ≥ 30 |
| 35%未満 | 劣位（赤） | n ≥ 30 |
| （n < 30 のとき） | 「参考値」バッジ＋警告文 | サンプル不足を明示 |

### 3.2 パターン統計エンジン（条件ビルダー）

ユーザーが条件を組み立てて過去30年を検索できる、本アプリの心臓部。

**条件の例（UIはフォームで組み立て）**:
- 「N日連続で下落」かつ「合計下落率がX%以上」→ 翌日の騰落
- 「前夜ダウがY%以上下落」かつ「ドル円が1円以上円高」→ 当日の騰落
- 「窓を開けて2%以上ギャップダウンで寄り付いた」→ 当日引けまでの騰落
- 「25日移動平均からの乖離率が-8%以下」→ 3日後までの騰落

**出力（必ずセットで表示）**:
1. 該当回数 n と発生日一覧
2. 翌日（または指定期間後）の上昇確率
3. 平均リターン / 中央値 / 最大上昇 / 最大下落
4. 95%信頼区間（例: 98.5% [92.0%–99.9%]）
5. 期間別の内訳（例: 1995–2005 / 2005–2015 / 2015–現在）— 「昔だけ通用したパターン」を見抜くため

**プリセットパターン**を最初から用意する:
- 連続下落からの反発（3日/5日/7日 × -5%/-10%/-20%）
- 大幅ギャップダウン/アップの寄り付き
- 移動平均乖離の極値
- 高ボラティリティ日（前日比±3%以上）の翌日

### 3.3 考慮要素のチェックボックス

統計の母集団を絞り込むフィルタとして機能する。ONにすると「その条件に該当する日だけ」で確率を再計算する。

**イベント・日程系**
- ☐ 日銀金融政策決定会合（当日/前日/翌日）
- ☐ FOMC（当日/翌日）
- ☐ 米雇用統計発表日（翌営業日）
- ☐ 米CPI発表日（翌営業日）
- ☐ メジャーSQ / マイナーSQ（当日/前日）
- ☐ 日本の祝日明け / 米国休場明け

**季節性・時期系（アノマリー）**
- ☐ 月別（1月効果、5月売り等の月別傾向）
- ☐ 曜日別
- ☐ 月初/月末（機関投資家のリバランス）
- ☐ 年度末・年度初（3月末/4月初）
- ☐ お盆・年末年始などの薄商い期間

**政治イベント系**
- ☐ 衆議院選挙（公示〜投開票、投開票翌日）
- ☐ 参議院選挙
- ☐ 米大統領選挙・中間選挙
- ☐ 自民党総裁選

- 各チェックボックスをONにした瞬間に **n（サンプル数）の変化を表示**する（絞り込みすぎでnが激減したら警告）
- イベント日程は過去分を静的データとして持ち、将来分はカレンダーデータを月次で更新する

### 3.4 事例ビューア

- 統計にヒットした過去事例を1件ずつチャートで確認できる（該当日を中心に前後20営業日）
- 「この事例の翌日はどうなった？」を伏せて表示 → クリックで開示、という予想→検証の見せ方も可能にする

### 3.5 通知（拡張フェーズ）

- 登録したパターン条件に現在の相場が合致したら、ブラウザ通知（Web Push）で知らせる
- 例: 「本日、"5日連続下落かつ-10%超" に合致しました。過去の翌日上昇確率: 82%（n=41）」

## 4. 予測ロジック仕様

### 4.1 MVP: ルールベース統計（ノンパラメトリック）

1. 現在の相場状態を特徴量ベクトル化する
   - 直近1/3/5/10/20日リターン、連続陽線/陰線数、移動平均乖離（5/25/75日）、
     前夜ダウ・NASDAQリターン、ドル円変化、ギャップ率、ボラティリティ（20日標準偏差）
2. 過去30年の全営業日から「似た状態の日」を抽出する
   - 方式A: ルール一致（条件ビルダーと同じ。解釈性が最高）
   - 方式B: k近傍法（特徴量空間で近い日を上位k件。ダッシュボードの自動判定に使用）
3. 抽出された日の「翌日リターン分布」をそのまま確率・期待値として表示する

### 4.2 拡張: 軽量ML（フェーズ3以降）

- 勾配ブースティング（LightGBM等）による翌日騰落の2値分類
- **必ずウォークフォワード検証**（学習期間を過去に固定し、未来のデータでテストを繰り返す）で精度を提示
- 特徴量重要度を開示し、ブラックボックス化を避ける
- MLの出力もダッシュボードでは「参考スコア」の1つとして統計エンジンと並列表示（MLを主にしない）

### 4.3 統計の誠実性ルール（実装で強制する）

これを守らないと「98.5%」のような数字が独り歩きして危険なため、仕様として固定する。

1. **n < 30 の確率には常に「サンプル不足」警告を付ける**
2. 確率には**必ず95%信頼区間を併記**する（Wilson区間）
3. **期間分割の再現性チェック**を自動表示（前半15年と後半15年で確率が大きく違うパターンは「不安定」ラベル）
4. 条件ビルダーで多数の条件を試すことによる**多重検定の問題**をヘルプで明示する
5. 手数料・スリッページ（先物ミニ想定で往復数百円＋1ティック）を控除した**実質期待値**も表示する

## 6. 技術スタック（提案）

| 層 | 技術 | 理由 |
|---|---|---|
| フロントエンド | React + TypeScript + Vite | SPA。チャートは Lightweight Charts（TradingView製・無料）または ECharts |
| バックエンド | Python (FastAPI) | pandas/numpy での統計計算と好相性。ML拡張も同一言語で可能 |
| DB | SQLite → PostgreSQL | 日足30年×数銘柄なら数万行程度。MVPはSQLiteで十分 |
| バッチ | Python スクリプト + cron | 1日2回のデータ取得・特徴量事前計算 |
| ホスティング | 個人利用ならVPS1台 or ローカル起動でも可 | 利用者が本人のみなら認証は簡易でよい |

- 日足データは全てブラウザに送っても数MB程度のため、**統計計算をフロント側で行う構成（バックエンドレス）も選択肢**。
  その場合はGitHub Pages等で完結できる（MVPを最速で作るならこちらを推奨）

## 8. 追加提案（あると良いもの）

1. **バックテスト機能**: 「このパターンで毎回翌日寄り買い・引け売りしていたら資産曲線はどうなったか」を1クリックで描画。確率だけでなく累積損益で判断できる
2. **ペーパートレード記録**: アプリの判定に従ったつもりの仮想売買を記録し、自分の運用成績として振り返れる（実弾投入前の検証に必須）
3. **レジーム分割ビュー**: バブル崩壊後/量的緩和期/アベノミクス以降/コロナ後 など金融環境で期間を分け、パターンの有効性が時代でどう変わったかを表示。「30年前の統計が今も通用するか」への答えになる
4. **NT倍率・騰落レシオなど市場内部指標**: 日経だけでなく市場の過熱感・広がりを条件に使えるようにする
5. **「今日の相場は過去のいつに似ているか」トップ5**: k近傍の結果をそのまま「似ている日ランキング」として見せる。直感的で説明力が高い
6. **損切り前提の期待値表示**: 「翌日上昇確率68%」だけでなく「-1.5%で損切りした場合の期待値」を出す。短期売買の実戦に近い数字になる
7. **ケリー基準の参考ポジションサイズ**: 確率と期待値から理論上の適正投入比率を参考表示（過大リスク防止のため上限キャップ付き）

## 9. 非機能要件

- 個人利用前提。認証はBasic認証程度（公開する場合は要検討）
- データ取得元の利用規約を遵守（yfinance等は個人利用の範囲で）
- 統計計算のレスポンス: 条件ビルダー実行から2秒以内（日足30年なら余裕）
- 全ての確率表示に免責文とn数を必ず添える（UIコンポーネントレベルで強制）
